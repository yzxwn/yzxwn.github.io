## HTTPS原理

#### Http

- web

```
web主要包括三个技术：
    http：超文本传输协议
    url：统一资源标识符
    html：超文本标记语言
```

- http

```
DNS协议解析出www.example.com的IP地址，然后连接服务器并发送请求
定义：HTTP提供了一组规则和标准，从而让信息能够在互联网上进行传播
语义：请求行、HTTP头部、响应行
特点：
    1. B/S模型：由客户端和服务器组成，通过网络层中的TCP协议进行通信，TCP能够提供一个可靠的、面向连接的传输服务
    2. 无状态：客户端通过Socket技术创建一个TCP/IP连接，并连接到服务器，完成信息交换后，关闭TCP连接，所有的HTTP请求/响应信息将全部消失，可通过Cookie和Session保存信息
    3. 跨平台：HTTP就是具备一定规则的纯文本信息，任何开发语言都可以实现HTTP或者基于HTTP进行开发
    4. 用途广泛：HTTP更多的是一个数据载体，对于客户端来说更重要的是如何处理这些数据
```

- 网络模型

```
HTTP是应用层协议，应用层协议是TCP/IP的一部分
TCP/IP：是标准的互联网网络协议
    分层：
        应用层：解释传输数据的含义
        传输层：TCP和UDP, TCP能够保证数据正确地到达，使用三次握手机制确保可靠传输，UDP不能保证数据正确传达，优点就是性能高减少了很多开销
        网络层：主要是IP这个协议，客户端和服务器传输的时候，会经过很多节点，IP就是选择一条最优的路径
        链路层：实体设备，包括光纤、网卡等设备，基于这些设备，数据最终才能到达终端
    封包/拆包机制：
        客户端封包，传输层接收到应用层消息后，在HTTP数据包前面增加TCP包头，然后发送给网络层；网络层在TCP数据包前面加上IP包头发送给链路层；链路层在IP数据包前面加上以太网包头；最终服务器接收到完整的数据包
        服务器拆包，首先在网络层去除链路层包头；在传输层去除IP包头；在应用层去除TCP包头；最终得到完整的HTTP应用层数据
Socket和TCP：TCP/IP需要通过Socket接口提供自身的能力，或者说Socket对TCP/IP进行了封装
```

- 互联网应用安全

```
互联网应用安全：HTTP本身的安全问题、Web应用安全问题
http协议不安全的根本原因：数据没有加密、无法验证身份、数据易篡改
Web应用安全：HTTP安全问题只是Web应用安全问题的一部分，就算网站支持了HTTPS, Web应用安全问题仍然很严峻
```

#### TLS

- TLS/SSL 协议

```
TLS协议是SSL协议的升级版，TLS/SSL协议代表TLS协议或者SSL协议
TLS/SSL协议是一系列算法的组合
TLS/SSL协议位于应用层协议和TCP之间，构建在TCP之上，由TCP协议保证数据传输的可靠性，任何数据到达TCP之前，都经过TLS/SSL协议处理
```

- HTTPS

```
HTTP和TLS/SSL协议组合在一起就是HTTPS, HTTPS拥有HTTP所有的特征，并且HTTP消息由TLS/SSL协议进行安全保护
TLS协议：握手层+加密层
```
