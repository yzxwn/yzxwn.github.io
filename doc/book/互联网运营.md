## 互联网运营

```
因为特定的时间有自带的热度会吸引更多的用户，所以对运营来说，完全可以在520之前上线一个活动页，而不是只把某次上线时间当做一个可以随便替换的时间
```

```
运营是感知并释放产品的独特魅力，使其连接用户的心理需求，满足用户体验的过程；运营是产品和用户之间的桥梁
```

```
阿里巴巴 /腾讯 /百度，他们研究的方向是把路拓得更宽，而不是宣传自己，你见过高速公路给自己打广告吗？
```

```
分工是不可逆转和阻挡的，我们唯一能做的，就是保持“眼高手低”的状态，在做好本职工作的同时也看看世界发生了什么变化，有什么新的工具诞生，有什么新的技能和知识可以学习，在奔跑中不断调整姿势、调整跑道
```

```
运营不是为了完成数据指标而存在的，运营可以让消费者认知产品，让产品部门重新认识产品，让企业看清产品。
运营代表了消费者，帮助用户在平凡中发现卓越，帮助消费者剔除不需要的功能、不需要的界面和不需要的产品。产品经理可以通过运营师的眼睛看清产品的不足，从而改进自己的产品
```

```
运营团队坚持不懈，克服万难，快速推广运营的是一款消费者不需要的产品。没有什么比这个更令人沮丧的了。
伪需求往往是某些创业者或产品经理把一些应用场景生搬硬套在某些人群上。创业者或产品经理经过简单的需求调查后就仓促上阵，有时用户说的需求不是真需求，如用户告诉你的需求是一把梯子，也许他真正的内心需求是上二楼。
判断用户需求通过分析用户行为要比让用户填写调查问卷更客观。
用户需求调查是倾听用户而不是听从用户，用户说的未必就是对的，用户说要买车可能只是想快速到达某个地点，并不是真的需要一辆车
```

```
有能把产品描绘得像花一样的老板未必是好团队，但有能把产品说得明明白白的员工一定是好团队
没听过见过哪个成功的互联网公司产品是靠外包技术团队开发的
```

```
烧钱购买用户习惯有很大的风险
补贴+补贴的前提是产品已经有清晰的商业模式，是在免费用户、付费用户和核心用户都具备的前提下进行的高难度动作
```

```
产品聚焦是为了让需求用户第一时间找到，并能轻易辨识
贴标签的目的也是为了节约消费者的时间
能在评论区给商品差评的用户一定程度上来说都是好用户，用户帮助商家找出产品缺陷对产品升级迭代提供了帮助
```

```
营销过程中给产品注入情愫，让产品有温度、有节操、有情怀。最后这些元素混合成一种叫诚意的东西，诚意是一张无形的包装纸，消费者喜欢追捧有诚意的产品。
消费者时代给我们的告诫是：你怎样对用户，用户就怎样对你。
```

```
企业也会被用户发好人卡：这个产品真好，但是我为什么要买？产品再好跟我有什么关系，我就是来看看
```

```
永远不要只做一个活动方案，一定要准备两个以上，让他人做选择而不是判断
```

```
活动：流程简单、降低门槛、用户体验安全、时效性（准时上下线）、活动页面（简单、流畅、兼容）
未经过测试就上线的活动是对公司和用户的极度不负责任，活动上线前一定要反复测试活动，保证活动在上线过程中的稳定运行，充分准备活动预案，设想活动中可能发生的情况，避免发生活动进行时用户迸发导致服务器崩溃的事件
开发需要：活动原型图（PC和移动版）、活动需求文档（设计文档、前后端开发文档）、活动流程图
```

```
先磨练职业，再扎根行业，有时黄金型专才要比通用型人才更吃香
```
